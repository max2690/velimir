# Анализ состояния сайта VELIMIR LUX

**Последнее обновление:** 26.01.2026

На текущий момент сайт представляет собой production-ready решение с продуманной архитектурой и современным UX. Ниже приведен актуальный разбор сильных и слабых сторон, а также рекомендации по SEO, визуалу и медиа.

---

## Как сайт выглядит (визуальная оценка)

### Сильные стороны
- **Стиль:** Строгая визуальная иерархия, «галерейное» настроение, архитектурная роскошь.
- **Цветовая палитра:** Глубокий графит (#0F1113), теплый белый (#EAE7E0), приглушенные акценты — единообразие и премиальность.
- **Типографика:** Cinzel (заголовки), Manrope, Inter — читаемость и характер бренда.
- **Hero:** Видео-фон с оверлеем, крупная типографика, V-образный переход в следующую секцию — сильный первый экран.
- **Секции:** Philosophy, Products (карусели с фото), Process, Cases (модалки), Gallery, FAQ, CTA — логичная структура и ритм.
- **Мобильная версия:** Полноэкранное модальное меню, адаптивная сетка, плавные анимации.
- **Детали:** Кастомный скроллбар, grain/V-overlay, параллакс в Gallery и Cases — ощущение «материальности».

### Что можно усилить визуально
- **Контраст CTA:** Кнопки «Обсудить проект» и форма — проверить достаточность контраста для доступности (WCAG).
- **Состояния фокуса:** Убедиться, что все интерактивные элементы имеют видимый focus ring для клавиатурной навигации.
- **Gallery/Cases без фото:** Пока нет реальных g01–g12 и c01–c03 — секции могут показывать заглушки или плейсхолдеры, чтобы не было битых картинок.

## SEO: что усилить

### Уже хорошо
- **Мета-теги:** В `layout.tsx` заданы `title`, `description`, `openGraph` (title, description, type, locale).
- **Семантика:** H1 на главной (VELIMIR), H2 по секциям, логичная иерархия.
- **SEO-страницы:** У каждой страницы в `(seo)/` свой `title` и `description`.
- **Техническая база:** `sitemap.xml`, `robots.txt`, `lang="ru"`, Next.js App Router (SSR/SSG).
- **Микроразметка:** FAQPage (Schema.org) на SEO-страницах через `SEOPageTemplate`.

### Рекомендации по усилению SEO

| Приоритет | Что сделать | Зачем |
|-----------|-------------|--------|
| **Высокий** | Добавить **og:image** (и по желанию **twitter:image**) в `layout.tsx` и на ключевые страницы | Корректный превью в соцсетях и мессенджерах, больше кликов из ссылок. |
| **Высокий** | Завести отдельное изображение для превью: например `/public/og-image.jpg` (1200×630 px). | Единый брендовый превью для всех страниц. |
| **Средний** | Добавить **canonical URL** для главной и SEO-страниц (через `metadataBase` и `alternates.canonical` в Next.js Metadata). | Избежание дублей в индексе, четкий основной URL. |
| **Средний** | Расширить **Open Graph** на SEO-страницах: свой `openGraph.title`, `openGraph.description`, `openGraph.url` (и при желании `openGraph.images`) для каждой. | Лучшие превью и релевантность при шаринге конкретных страниц. |
| **Средний** | Добавить **Schema.org Organization** в `layout.tsx` (название, url, logo, контакты, телефон). | Поддержка знаний о бренде и локального поиска. |
| **Низкий** | Мета **keywords** — по желанию (поисковики почти не используют). | Минимальный плюс для части запросов. |
| **Низкий** | Более описательные **alt** у изображений: не только «Velimir Work 1», а «Стол из эпоксидной смолы и дуба в интерьере» и т.п. | Доступность и контекст для поиска по картинкам. |
| **Низкий** | Для страниц продуктов/услуг рассмотреть разметку **Product** или **Service** (Schema.org). | Возможность расширенных сниппетов в выдаче. |

### Технические моменты
- **Домен в sitemap/robots:** Сейчас указан `https://velimir-lux.ru` — проверить соответствие реальному домену после деплоя.
- **Скорость:** Использование `next/image` и оптимизированные изображения продуктов уже помогают Core Web Vitals.

## Слабые места (точки роста)

### 1. Визуальный контент (Assets)
- **Продукты:** Все 12 изображений загружены и оптимизированы.
- **Hero:** Видео и постер загружены.
- **Галерея:** Ожидается загрузка 12 работ (g01.jpg … g12.jpg).
- **Кейсы:** Ожидается минимум 3 изображения (c01.jpg … c03.jpg).
- **QR-код:** Ожидается создание и загрузка.
- **Рекомендация:** Добавить og-image и favicon для полноты SEO и бренда.

### 2. Инфраструктура Rate Limit
- Текущий Rate Limit — in-memory; при Serverless (Vercel) или нескольких инстансах возможны расхождения. Для масштабирования — перенос в Redis (например, Upstash).

### 3. Blur-up placeholders
- Настроить `blurDataURL` для изображений в Products, Gallery, Cases — плавное появление при загрузке.

### 4. Индикатор прогресса скролла
- Тонкая полоска прогресса под Header улучшит UX длинной главной страницы.

### 5. Обработка ошибок API
- Критические ошибки API логируются в консоль; целесообразно дублировать в Telegram-бот администратора.

### 6. Префиксные сообщения для мессенджеров
- В ссылках WhatsApp/Telegram добавить `?text=Здравствуйте, хочу обсудить проект` для повышения конверсии.

### 7. A/B тестирование Hero
- После накопления трафика — тест вариантов заголовка через Яндекс.Метрику.
