- **Отказоустойчивость:** Дублирование лидов (YouGile + Email) минимизирует риск потери клиента.
- **Мобильный UX:** ✅ Реализовано полноэкранное модальное меню с плавными анимациями.
- **Интерактивность:** ✅ Добавлены эффекты Grain (заменен на V-тень) и мягкий параллакс для создания глубины.
- **Аналитика:** ✅ Настроено отслеживание глубины скролла, времени на странице и взаимодействий с продуктами.

## Слабые места (Точки роста)

### 1. Визуальный контент (Assets) ⚠️ ЧАСТИЧНО РЕШЕНО
- ✅ **Продукты:** Все 12 изображений продуктов загружены и оптимизированы (снижение веса на 90-95%).
- ✅ **Hero видео:** Видео и постер загружены.
- ⏳ **Галерея:** Ожидается загрузка 12 работ (g01.jpg ... g12.jpg).
- ⏳ **Кейсы:** Ожидается загрузка минимум 3 изображений (c01.jpg ... c03.jpg).
- ⏳ **QR-код:** Ожидается создание и загрузка QR-кода.
- **Приоритет:** Средний. Основной контент готов, осталось дополнить галерею и кейсы.

### 2. Инфраструктура Rate Limit
- **Проблема:** Текущий Rate Limit работает "в памяти" (in-memory).
- **Риск:** При использовании Serverless-хостинга (Vercel) или нескольких инстансов сервера лимиты будут работать некорректно. Нужно решение на базе Redis.
- **Приоритет:** Средний. Для MVP текущее решение достаточно, но для масштабирования потребуется Redis.

### 3. Blur-up Placeholders для изображений
- **Проблема:** Изображения загружаются без плавного перехода (нет blurDataURL).
- **Рекомендация:** Настроить `blurDataURL` для всех изображений через `next/image`, чтобы они плавно "проявлялись" при загрузке.
- **Приоритет:** Низкий. Улучшает восприятие, но не критично.

### 4. Индикатор прогресса скролла
- **Проблема:** Нет визуального индикатора прогресса чтения страницы.
- **Рекомендация:** Добавить очень тонкую полоску прогресса в верхней части экрана (под Header).
- **Приоритет:** Низкий. Nice-to-have для улучшения UX.

### 5. Обработка ошибок API
- **Проблема:** Критические ошибки API логируются только в консоль сервера.
- **Рекомендация:** Настроить отправку критических ошибок в Telegram-бот администратора для быстрого реагирования.
- **Приоритет:** Средний. Важно для мониторинга в production.

### 6. Префиксные сообщения для мессенджеров
- **Проблема:** Ссылки на WhatsApp/Telegram открывают пустой чат.
- **Рекомендация:** Добавить префиксные сообщения в URL (например: `https://wa.me/70000000000?text=Здравствуйте, хочу обсудить проект`).
- **Приоритет:** Низкий. Улучшает конверсию, но не критично.

### 7. A/B тестирование Hero заголовка
- **Проблема:** Нет данных о том, какой заголовок лучше конвертирует.
- **Рекомендация:** Подготовить два варианта заголовка и настроить A/B тест через Яндекс.Метрику.
- **Приоритет:** Низкий. Можно делать после запуска и сбора первых данных.
